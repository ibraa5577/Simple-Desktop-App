Social Network Application

Description: A Brief Overview

This project is about s JavaFX application for manging user profiles. It allows users to create, delete, and modify profiles, including changing profile pictures and status messages. Moreover, users can add friends to thier profiles. The application provides a graphical user interface (GUI) for interacting with profile features.

Design Overview

FXApplication Class

  • Purpose:

    • The FXApplication class is the main class that extends Application from JavaFX.
    • It represents the GUI of the application.

  • Variables:

    • borderPane: Represents the main layout of the GUI.
    • topHbox: Represent the upper layout of the GUI.
    • leftVbox: Represent the left layout of the GUI.
    • nameTextField: Text field for user input (name).
    • imagePathTextFiled: Text field for the reletive image path.
    • changeStatusTextField: Text field for usetr statues.
    • addFriendTextFiled: Text field for friend userName.
    • addButton: Button to add a profile.
    • deleteButton: Button to delete a profile.
    • lookupButton: Button to show a user profile.

JavaFX Application - start Method Explanation

The start method is a part of the FXApplication class that extends Application.

Overview:

  • Scene Setup:

    • A BorderPane named borderPane is created to organize the layout.
    • The background color is set to light gray.

  • Top Region (HBox):

    • An HBox named topHBox is created for the top region.
    • It contains UI elements such as a Label (nameLabel), a TextField (nameTextField), and buttons (addButton, deleteButton, lookupButton).
    • A ComboBox (themeComboBox) for selecting themes is also added.

  • Left Region (VBox):

    • A VBox named leftVBox is created for the left region.
    • It contains three sub-VBoxes (subVBox1, subVBox2, subVBox3), each with a TextField and a related Button.
    • These are used for changing status, changing pictures, and adding friends.

  • Event Handling:

    • Event handlers are attached to buttons to perform actions like adding a profile, deleting a profile, and changing a picture.

Usage:

  • Interact with the elements to add/delete profiles, change pictures/status, add friends, and look up user information.

addProfile Method Explanation

The addProfile method is responsible for adding a new user profile.

Overview:

  • User Input:

    • Retrieves the new profile name from the nameTextField input field.

  • Profile Creation:

    • Creates an instance of the Profile class with the provided name.
    • Calls the creatProfile method of the Profile class, attempting to create a new user profile folder.

  • Show Center:

    • Checks if the provided name is not empty and the profile creation was successful.
    • If conditions are met, calls the showCenter method with the argument “add” to display a message indicating that the profile has been added successfully.

Usage:

  • Enter a name for the new profile in the nameTextField.
  • Click the “Add” button to trigger the addProfile method.
  • If the profile is created successfully, a message confirming the addition will be displayed in the center of the application.

deleteProfile Method Explanation

The deleteProfile method serves the purpose of deleting an existing user profile.

Overview:

  • User Input:

    • Retrieves the name of the profile to be deleted from the nameTextField input field.

  • Profile Existence Check:

    • Creates an instance of the Profile class with the provided name.
    • Checks if the user profile already exists.

  • Profile Deletion:

    • If the profile exists, calls the deleteProfile method of the Profile class to delete the user profile folder.

  • Show Center:

    • If the profile exists and is successfully deleted, calls the showCenter method with the argument “delete” to display a message indicating that the profile has been deleted.

Usage:

  • Enter the name of the profile you want to delete in the nameTextField.
  • Click the “Delete” button to trigger the deleteProfile method.
  • If the profile exists and is deleted successfully, a message confirming the deletion will be displayed in the center of the application.

addFriends Method Explanation

The addFriends method is responsible for adding a friend to a user’s friend list.

Overview:

  • User Input:

    • Retrieves the name of the current user from the nameTextField input field.
    • Retrieves the friend’s username from the addFriendTextFiled input field.

  • Validation:

    • Checks if the friend’s username is not empty before attempting to add friends.

  • Profile Existence Check:

    • Creates an instance of the Profile class for the current user.
    • Verifies that the current user’s profile folder exists and that the friend’s username is not the same as the current user.

  • Add Friends:

    • Calls the friends method of the Profile class to add the friend to the current user’s friend list.
    • Prints a message to the console indicating the successful addition of the friend.

  • Show Center:

    • Calls the showCenter method with the argument “add friend” to display a message indicating that the friend has been added successfully.

Usage:

  • Enter the name of the current user in the nameTextField.
  • Enter the friend’s username in the addFriendTextFiled.
  • Click the “Add Friends” button to trigger the addFriends method.
  • If the friend’s username is not empty, the current user’s profile exists, and the friend is added successfully, a message confirming the addition will be displayed in the center of the application.

showUserInfo Method Explanation

The showUserInfo method retrieves and displays user information.

Overview:

  • User Input:

    • Gets the name of the current user from nameTextField.

  • Profile Handling:

    • Creates a Profile instance for the current user.
    • Retrieves files (friends, status, image, name) from the user’s profile.

  • Enter the user’s name in nameTextField.
  • Click “Lookup” to display user info in the app.
  • Handles scenarios with and without a profile.

Profile Class Overview

The Profile class designed to manage user profiles within a Java application. It includes methods for creating, deleting, and updating user profiles, handling user images, saving user status, managing friends, and retrieving user information.

Fields

  • name: Represents the username associated with the profile.
  • folder: Represents the directory where user-specific data is stored.

Methods

Profile(String name)

  • Constructor: Initializes a Profile object with the provided username (name).

boolean createUserProfile()

  • Description: Creates a new user profile folder and saves the username in a corresponding file.
  • Returns: true if the profile is successfully created, false otherwise.

void createImage(String imagePath)

  • Description: Moves an image file to the user’s profile folder, overwriting existing images.
  • Parameters:
    • imagePath: The path to the source image file.

void deleteProfile()

  • Description: Deletes the user profile folder and its contents.

void saveStatus(String status)

  • Description: Saves the user’s status in a file within the user’s profile folder.
  • Parameters:
    • status: The status to be saved.

boolean userProfileExists()

  • Description: Checks if the user profile folder exists.
  • Returns: true if the profile folder exists, false otherwise.

void friends(String currentUser, String friendUser)

  • Description: Add a friend between a user and another by updating their friend lists.
  • Parameters:
    • currentUser: The usage username.
    • friendUser: The friend’s username.

File[] getUserFiles()

  • Description: Retrieves an array of File objects representing the contents of the user’s profile folder.
  • Returns: An array of File objects or null if the user profile doesn’t exist.

Note

  • The project structure includes a “data” directory inside the “src” directory for storing user data. Evrey user has his own folder within his information.
